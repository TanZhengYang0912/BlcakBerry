# BlackBerry

A collection of Python security and threat intelligence tools built to learn API integration, encryption, and local AI enrichment.

---

## Tools

### Threat Intelligence Dashboard
Fetches the 5 most recent threat pulses from [AlienVault OTX](https://otx.alienvault.com) and enriches each one with an AI-generated summary and severity rating (High / Medium / Low) using a locally-running Ollama model — no cloud AI costs, no data sent externally.

Two interfaces available:
- **`final.py`** — Flask web app with colour-coded threat cards in the browser
- **`gui.py`** — Tkinter desktop GUI with live per-pulse progress tracking

**`test.py` / `test2.py`** — CLI scripts used during development to test OTX fetching and Ollama enrichment independently.

### Password Generator (`Password_generation.py`)
Generates a random password using letters, digits, and symbols. Minimum length of 8 characters enforced.

```
python Password_generation.py
```

### WHOIS Lookup (`Testwhois.py`)
Performs a WHOIS lookup on any domain and prints registrar, creation/expiry dates, name servers, and contact info.

```
python Testwhois.py
```

### Encryption Tool (`encryption_script.py`)
Symmetric encryption and decryption using the [Fernet](https://cryptography.io/en/latest/fernet/) algorithm. Generates a secret key stored in `secret.key`.

```
python encryption_script.py
```

## Setup

### 1. Install dependencies

```
pip install -r requirements.txt
```

### 2. Configure API key

Create a `.env` file in the project root:

```
OTX_API_KEY=your_otx_api_key_here
```

Get your free API key at [otx.alienvault.com](https://otx.alienvault.com) → Settings → API Key.

### 3. Install and run Ollama

Download Ollama from [ollama.com](https://ollama.com), then pull the model:

```
ollama pull phi3
```

### 4. Run

```bash
# Web dashboard
python final.py

# Desktop GUI
python gui.py
```

---

## Notes

- `.env` is excluded from version control via `.gitignore` — never commit your API key
- Ollama runs entirely on localhost; no threat data is sent to external AI services
- `secret.key` (generated by `encryption_script.py`) should also not be committed
